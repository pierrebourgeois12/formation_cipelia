Bienvenue! Ceci est un MCP minimaliste pour apprendre comment ça marche.

═══════════════════════════════════════════════════════════════════════════════
⚡ INSTALLATION (2 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

1. Créer l'environnement virtuel:
   
   python -m venv venv
   source venv/bin/activate    # Mac/Linux
   venv\Scripts\activate       # Windows


2. Installer les dépendances:
   
   pip install -r requirements.txt


═══════════════════════════════════════════════════════════════════════════════
🧪 TESTER EN LOCAL (3 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

OPTION A: Lancer le serveur (voir qu'il démarre)

   python -m src.main

✅ Vous verrez: 
   🚀 Serveur MCP Formation démarré
   Outils disponibles: hello, calculate, list_resources, read_resource

L'erreur JSON qui suit est NORMALE - c'est le serveur qui attend un client MCP.
Arrêter: Ctrl+C


OPTION B: Tester les outils (RECOMMANDÉ)

   pytest tests/ -v

Vous verrez les résultats de 7 tests des tools.
C'est la meilleure façon de vérifier que tout fonctionne!


═══════════════════════════════════════════════════════════════════════════════
🤖 TESTER SUR CLAUDE DESKTOP (2 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

OPTION A: Auto-configuration (Windows)

   python setup_claude_config.py
   # Puis relancez Claude Desktop


OPTION B: Configuration manuelle

   1. Ouvrez: %APPDATA%\Claude\claude_desktop_config.json
   
   2. Ajoutez:
   {
     "mcpServers": {
       "mcp-formation": {
         "command": "python",
         "args": ["-m", "src.main"],
         "cwd": "C:\\chemin\\vers\\mcp-formation"
       }
     }
   }
   
   3. Relancez Claude Desktop

═══════════════════════════════════════════════════════════════════════════════
📚 CONCEPTS CLÉS
═══════════════════════════════════════════════════════════════════════════════

1. TOOLS (Outils)
   ┌─────────────────────────────────────────────────────┐
   │ @mcp.tool()                                         │
   │ async def hello(name: str) -> str:                  │
   │     return f"Bonjour {name}!"                       │
   └─────────────────────────────────────────────────────┘
   Claude peut appeler ces outils directement.


2. RESOURCES (Ressources)
   ┌─────────────────────────────────────────────────────┐
   │ @mcp.resource("guide://formation")                 │
   │ async def get_guide() -> str:                       │
   │     return "Contenu du guide..."                    │
   └─────────────────────────────────────────────────────┘
   Claude peut accéder à ces données structurées.


3. PROMPTS (Instructions)
   ┌─────────────────────────────────────────────────────┐
   │ Tu es un expert en mathématiques.                   │
   │ Utilise calculate() pour les calculs.               │
   │ Sois pédagogique.                                   │
   └─────────────────────────────────────────────────────┘
   Guide le comportement de Claude.


═══════════════════════════════════════════════════════════════════════════════
🎯 UTILISER AVEC CLAUDE
═══════════════════════════════════════════════════════════════════════════════

Une fois configuré sur Claude:

1. Appeler un outil:
   "Appelle hello avec name='Formation'"
   → Claude appellera hello("Formation")
   → Vous recevrez le résultat


2. Accéder à une ressource:
   "Lis la ressource example.txt"
   → Claude appellera read_resource("example.txt")


3. Utiliser un prompt:
   [Copiez le contenu de prompts/expert_math.md en début de convo]
   "Calcule 15 * 27"
   → Claude utilisera le context du prompt
   → Claude appellera calculate("multiply", 15, 27)


═══════════════════════════════════════════════════════════════════════════════
💡 EXEMPLE: AJOUTER UN OUTIL
═══════════════════════════════════════════════════════════════════════════════

1. Ouvrir src/main.py

2. Ajouter:
   
   @mcp.tool()
   async def my_tool(param: str) -> str:
       """Description de l'outil"""
       return f"Résultat: {param}"


3. Tester localement:
   
   python -m src.main


4. Utiliser dans Claude:
   
   "Appelle my_tool avec param='test'"

❓ QUESTIONS FRÉQUENTES
═══════════════════════════════════════════════════════════════════════════════

Q: Quand je lance python -m src.main, j'ai une erreur JSON?
R: C'est NORMAL! Le serveur démarre correctement. L'erreur vient du fait qu'il
   attend un client MCP (Claude) pour communiquer. Utilisez à la place:
   - pytest tests/ -v  (pour tester les outils)
   - Claude Desktop    (pour utiliser les outils)

Q: Comment vérifier que le serveur fonctionne?
R: Lancer: pytest tests/ -v
   Si les 7 tests passent, tout est OK!

Q: Claude ne voit pas mon MCP?
R: Relancez Claude COMPLÈTEMENT (fermez et réouvrez)
   Vérifiez le chemin dans claude_desktop_config.json

Q: Comment ajouter un outil?
R: Dans src/main.py, utilisez @mcp.tool() avant votre fonction

Q: Comment accéder à une ressource?
R: Créez un fichier dans resources/, puis utilisez read_resource()

Q: Comment guider Claude avec un prompt?
R: Copiez le contenu d'un fichier prompts/ au début de votre conversation

Q: Ça marche sur autres modèles?
R: Les MCPs fonctionnent avec n'importe quel modèle Claude
